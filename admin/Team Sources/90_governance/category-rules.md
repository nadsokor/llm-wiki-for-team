# Category Rules

Source classification:

- `product`: PRD, roadmaps, specs, feature proposals, launch plans
- `engineering`: architecture notes, technical plans, API docs, incident reviews
- `research`: user research, market research, interviews, external references
- `process`: meeting notes, decisions, rituals, workflow documentation

Public wiki classification:

- `decisions`: explicit product, engineering, or process decisions
- `concepts`: stable internal concepts or shared definitions
- `glossary`: normalized term-definition pairs
- `maps`: curated overview pages that organize multiple ground-truth notes

Escalation rules:

- weak evidence stays in `10_sources/` only
- strong or repeated consensus may be promoted into `20_public_wiki/`
- conflicting evidence should trigger an update or supersession note, not duplicate ground truth
